v2.0 ПОСЛЕДНИЙ

2026-05-13

Multi-tenant foundation + Docker deploy — крупный архитектурный сдвиг. Несколько компаний на одной инстансе с резолвингом по subdomain'у; zero-downtime upgrades через scripts/update.sh.

Multi-tenant routing

  • companies.subdomain + TenantResolver middleware: acme.hrms.example.com → Current.company, fallback на первую компанию на apex-домене.
  • Current ActiveSupport::CurrentAttributes (company, user) + helper current_company.
  • Настройки → Коммуникации → 🏢 Subdomain — superadmin задаёт tenant slug.
  • 404 при неизвестном subdomain (JSON для API, HTML для браузера). www / api / app исключены.

Развёртывание

  • scripts/update.sh — бэкап → fetch → миграции на новом образе до swap'а → rolling restart → healthcheck-цикл → rollback-подсказка при failure.
  • docker-compose.prod.yml override: pre-built image, healthcheck, JSON-логи с rotation, PG tuning (shared_buffers 512MB, work_mem 16MB), Puma 3×10.
