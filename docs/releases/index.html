v2.0 LATEST

2026-05-13

Multi-tenant foundation + Docker deploy — major architectural shift. Run multiple companies on one HRMS instance with subdomain routing; zero-downtime upgrades via scripts/update.sh.

Multi-tenant routing

  • companies.subdomain column + TenantResolver middleware: acme.hrms.example.com → Current.company, fallback to first company on apex domain.
  • Current ActiveSupport::CurrentAttributes (company, user) + current_company helper.
  • Settings → Communications → 🏢 Subdomain — superadmin sets tenant slug.
  • 404 for unknown subdomain (JSON for API, HTML for browser). www / api / app excluded.

Deployment

  • scripts/update.sh — backup → fetch → migrate on new image before swap → rolling restart → healthcheck loop → rollback hint on failure.
  • docker-compose.prod.yml override: pre-built image, healthcheck, JSON-logging with rotation, PG tuning (shared_buffers 512MB, work_mem 16MB), Puma 3×10.
Full notes on GitHub → Install instructions
