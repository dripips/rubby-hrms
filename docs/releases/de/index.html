v2.0 NEUESTER

2026-05-13

Multi-Tenant-Grundlage + Docker Deploy — großer architektonischer Schritt. Mehrere Firmen auf einer HRMS-Instanz mit Subdomain-Routing; Zero-Downtime-Updates via scripts/update.sh.

Multi-Tenant-Routing

  • companies.subdomain + TenantResolver-Middleware: acme.hrms.example.com → Current.company, Fallback auf erste Firma bei Apex-Domain.
  • Current ActiveSupport::CurrentAttributes (company, user) + current_company-Helper.
  • Einstellungen → Kommunikation → 🏢 Subdomain — Superadmin setzt Tenant-Slug.
  • 404 bei unbekannter Subdomain. www / api / app ausgeschlossen.

Deployment

  • scripts/update.sh — Backup → Fetch → Migrationen auf neuem Image vor Swap → Rolling Restart → Healthcheck-Loop → Rollback-Hinweis bei Fehler.
  • docker-compose.prod.yml Override: pre-built Image, Healthcheck, JSON-Logging mit Rotation, PG-Tuning, Puma 3×10.
Vollständige Notes auf GitHub → Installationsanleitung
