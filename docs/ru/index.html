Зачем ещё одна HRMS?

Большинство HR-систем зашивают словарь одной индустрии. Найм — это «разработчик», отпуск — это «ежегодный», документ — это «паспорт». Реальные компании сложнее: септик-сервису нужны категории ADR-допусков, клинике — номера лицензий, IT-стартапу — ссылки на GitHub.

HRMS подстраивается под компанию, а не наоборот. Каждая сущность поддерживает Custom Fields через словари. HR описывает схему один раз — формы, валидация, AI-разбор и аудит подхватывают её автоматически.

Установка за 2 минуты

Одна bash-команда. Установщик генерит секреты, билдит образ, поднимает db + app + worker, создаёт superadmin'а, печатает доступы.

git clone https://github.com/dripips/rubby-hrms.git
cd rubby-hrms
./scripts/install.sh

Обновление позже — zero-downtime, с автобэкапом:

./scripts/update.sh              # последний master
./scripts/update.sh v2.0         # конкретный тег

🐳 Production

Pre-built образ из registry, healthcheck, PG-tuning, JSON-логи с rotation.

docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d

🧑‍💻 Development

Ruby 4.0.3 + PostgreSQL 18. Без Docker.

bundle install && bin/rails db:setup db:seed && bin/dev
