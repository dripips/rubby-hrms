Why another HRMS?

Most HR systems hardcode one industry's vocabulary. A hire is a "developer", a leave is "PTO", a document is a "passport". Real companies are messier — a septic service tracks ADR clearances; a clinic tracks license numbers; a tech startup wants GitHub URLs.

HRMS bends to the company, not the other way around. Every entity has a Custom Fields system through Dictionaries. HR defines the schema once — forms, validation, AI extraction, and audit pick it up automatically.

Install in 2 minutes

One bash command. The installer generates secrets, builds the image, starts db + app + worker, creates a superadmin, prints the credentials.

git clone https://github.com/dripips/rubby-hrms.git
cd rubby-hrms
./scripts/install.sh

Upgrade later — zero-downtime, with automatic backup:

./scripts/update.sh              # latest master
./scripts/update.sh v2.0         # specific tag

🐳 Production

Pre-built image from registry, healthcheck, PG tuning, JSON logging with rotation.

docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d

🧑‍💻 Development

Ruby 4.0.3 + PostgreSQL 18. No Docker required.

bundle install && bin/rails db:setup db:seed && bin/dev
