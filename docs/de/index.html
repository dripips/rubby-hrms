Warum noch ein HR-System?

Die meisten HR-Systeme hartcodieren das Vokabular einer Branche. Ein Mitarbeiter ist ein „Developer", ein Urlaub ist „Jahresurlaub", ein Dokument ist ein „Reisepass". Echte Unternehmen sind komplizierter — ein Klempnerdienst braucht ADR-Zulassungen, eine Klinik braucht Lizenznummern, ein Tech-Startup will GitHub-URLs.

HRMS passt sich dem Unternehmen an, nicht umgekehrt. Jede Entität unterstützt Custom Fields über Dictionaries. HR definiert das Schema einmal — Formulare, Validierung, KI-Extraktion und Audit übernehmen es automatisch.

KI-Agenten — für jede Phase des Mitarbeiter-Lebenszyklus

In 2 Minuten installieren

Ein Bash-Befehl. Der Installer generiert Secrets, baut das Image, startet db + app + worker, erstellt einen Superadmin, druckt die Zugangsdaten.

git clone https://github.com/dripips/rubby-hrms.git
cd rubby-hrms
./scripts/install.sh

Upgrade später — Zero-Downtime, mit automatischem Backup:

./scripts/update.sh              # neuester master
./scripts/update.sh v2.0         # spezifischer Tag

🐳 Production

Pre-built Image aus Registry, Healthcheck, PG-Tuning, JSON-Logging mit Rotation.

docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d
